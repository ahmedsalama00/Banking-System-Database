## 🛣️ My Data Journey: Path to Mastery
This project bridges the gap between **Data Engineering** and **Data Analytics**:

### 1. The Blueprint (Design Phase) 📐
* **Relational Mapping**: Created a multi-table architecture (`Customers`, `Accounts`, `Transactions`, `Loans`, `Cards`, `Employees`, `Branches`, `Transfers`, `Loan_Payments`).
* **Data Integrity**: Implemented Primary/Foreign keys and constraints to ensure a "Single Source of Truth" [cite: 1, 2, 8, 13, 21].

### 2. The Engine (Query Phase) ⚙️
I developed a hierarchical query system to demonstrate progressive mastery:
* **Basics**: Selection, filtering (e.g., finding customers by birth year or city) [cite: Queries.sql].
* **Intermediates**: Aggregate functions (AVG, SUM, MAX/MIN) and Grouping for branch-level reporting [cite: Queries.sql].
* **Advanced**: Multi-table Joins and Subqueries to identify "Pro-level" insights, like finding customers with balance higher than the global average or identifying high-earning employees [cite: Queries.sql].

### 3. The Insight (Analysis Phase) 📈
* **Business Intelligence**: Converting raw rows into KPIs like "Total Loan Amounts" and "Transaction Frequencies" [cite: Queries.sql].
* **Customer Segmentation**: Identifying active vs. inactive users and credit card owners [cite: Queries.sql].
